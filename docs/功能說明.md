# Heimdall 功能說明（繁體中文）

這是一份「給使用者看」的功能總覽，逐一說明這個工具目前能做什麼、怎麼用、以及每個畫面背後的意義。
寫作目標是**好懂**：每個功能都用「用途 → 怎麼用 → 你會看到什麼」三段說明。

> 目前進度：Phase 0–6 已完成，**8 大分析師視角全部到位，且台股（`.TW`/`.TWO`）已支援**。可用功能 =
> 11 個網頁分頁（選股、個股圖、基本面、技術面、風險、財報、回測、多因子、總經、產業輪動、ETF 配置），
> 左側欄可切換 **English / 繁體中文**。台股股價走 yfinance（已還原），台股財報與**月營收**走 FinMind
> （免費、免金鑰；設 `FINMIND_TOKEN` 可提高額度）。其中「總經」需免費的 `FRED_API_KEY`、「財報」（美股
> 共識預估）需付費的 `FMP_API_KEY`，沒設定時會顯示提示、其他功能照常。

---

## 0. 這是什麼？一句話版本

一個**自己用的選股 + 進出場判斷 + 策略回測**網站。資料目前用**免費來源**（美股股價＝yfinance、
美股財報＝美國證交會 SEC EDGAR、總經數據＝FRED）。所有數字都「**據實以告**」——回測一定計入手續費與
滑價、用隔日開盤價成交，不會給你過度美化的結果。

## 1. 快速開始（三個指令）

工具用 [`uv`](https://docs.astral.sh/uv/) 管理，已綁定 Python 3.12（電腦預設的 conda 3.8 不會被動到）。

```bash
# 1) 安裝環境（第一次）
uv sync --all-extras

# 2) 先建立「選股快照」（選股器 / 多因子頁面需要它；建議先在 .env 設定 SEC_EDGAR_USER_AGENT）
uv run python -m heimdall.screener.build              # 美股
uv run python -m heimdall.screener.build --market tw  # 台股（2330.TW 等）

# 3) 開啟網站（瀏覽器）
uv run streamlit run src/heimdall/ui/app.py
```

打開網站後，左側欄最上方可切換語言（English / 繁體中文）與分頁。**台股用法**：在「個股圖、技術面、
風險、回測、基本面」等頁面，代號輸入 `2330.TW`（上市）或 `xxxx.TWO`（上櫃）即可；基本面頁在台股會
多顯示一塊**月營收**走勢與年增率。下面逐一說明各分頁。

## 2. 六大分頁功能

### 📊 選股器（Screener）

- **用途**：用你自訂的條件，從一籃子股票中篩出符合的標的。
- **怎麼用**：
  1. 先跑過上面的「建立選股快照」指令（否則畫面會提示你去建立）。
  2. 從「預設條件」挑一個起手式，或載入你存過的條件：
     - **Cheap & profitable（便宜又賺錢）**：本益比 < 25、ROE > 15%、淨利率 > 10%
     - **Oversold quality（超賣的好公司）**：RSI < 40、營收年增 > 5%
     - **Above 200-day trend（站上年線）**：股價高於 200 日均線
  3. 在表格裡**自由增刪條件**：每一列是「欄位 / 運算子 / 數值」，例如 `pe < 15`、`rsi_14 < 30`。
  4. 設定「排序欄位、升冪/降冪、顯示幾檔」，按下後即時看到結果。
  5. 可把這組條件**存起來**（之後在「載入」下拉選單重用）。
- **你會看到什麼**：一張符合條件的股票表格（含本益比、ROE、報酬率…等欄位）。
- **重要觀念**：**缺資料 = 直接淘汰，不會偷偷放行**。例如某檔沒有本益比，套用「本益比 < 15」時它會被排除，
  而不是被當成通過。
- **可篩選的欄位（部分）**：`pe`（本益比）、`ps`（股價營收比）、`fcf_yield`（自由現金流殖利率）、
  `net_margin`（淨利率）、`gross_margin`（毛利率）、`roe`、`debt_to_equity`（負債/權益）、
  `revenue_growth_yoy`（營收年增）、`rsi_14`、`ret_3m`/`ret_6m`/`ret_12m`（近 3/6/12 月報酬）、
  `pct_above_sma_200`（高於年線幅度）、`market_cap`（市值）。

### 📈 個股圖表（Chart）

- **用途**：看單一股票的 K 線與技術指標。
- **怎麼用**：輸入代號（格式 `代號.市場`，例如 `AAPL.US`、未來台股 `2330.TW`），拉「回看天數」滑桿。
- **你會看到什麼**：互動式圖表，分三層——
  1. **K 線 + 三條均線**（20/50/200 日移動平均線）
  2. **RSI(14)**，含 30 / 70 參考線
  3. **MACD**（快線、訊號線、柱狀圖）

### 🏛 基本面儀表板（Fundamental，高盛視角）

- **用途**：像券商研報那樣，快速體檢一家美股公司的基本面。
- **怎麼用**：輸入美股代號（資料來自 SEC EDGAR，僅限有在美國申報的公司）。
- **你會看到什麼**：
  1. **評級摘要框**：評等（Buy/Hold/Sell）、分數（0–100）、本益比、股價營收比、營收年複合成長率（CAGR）。
     > 評等是**用公開規則算出來的**（毛利/淨利率、成長、負債、自由現金流、估值各佔一份），不是猜的。
  2. **歷年圖**：各會計年度的營收長條圖、毛利/營業/淨利率折線圖。
  3. **多空對照**：自動列出「看多理由」與「看空理由」（例如「高淨利率」「估值偏貴 P/E 36」）。
  4. **情境價位**：用 15× / 22× / 30× 本益比乘上每股盈餘的「示意」區間（標明僅供參考）。
  5. **（選用）AI 研究報告按鈕**：見第 4 節。

### 📐 技術面儀表板（Technical，摩根士丹利視角）

- **用途**：把零散的技術訊號整理成一份「**交易計畫**」。
- **怎麼用**：輸入代號即可。
- **你會看到什麼**：
  1. **交易計畫摘要框**：目前價、進場價、停損價、第一目標價（1R）、短/中/長期趨勢。
  2. **指標列**：RSI(14)、ATR(14)、布林帶 %B、近期是否出現黃金/死亡交叉。
  3. **圖表**：K 線 + 布林通道 + 支撐/壓力線 + 停損/目標線。
  4. **表格**：支撐/壓力價位、斐波那契回撤位。
  5. **（選用）AI 交易計畫報告按鈕**。
- **停損/目標怎麼算**：以 ATR（波動度）為基礎——停損 = 進場 − N×ATR；目標 = 進場 + 風險的 1R/2R/3R 倍數。

### 🧪 策略回測（Backtest）

- **用途**：驗證一個「進出場策略」在歷史上的表現，並做參數測試。
- **怎麼用**：
  1. 選代號、歷史年數。
  2. 選**策略**：
     - **SMA crossover（均線交叉）**：快線向上穿越慢線買進，向下穿越賣出。
     - **Donchian breakout（唐奇安突破）**：突破前 N 日高點買進，跌破前 M 日低點賣出。
     - **RSI mean-reversion（RSI 均值回歸）**：RSI 跌破超賣區買進，升破超買區賣出。
  3. 調策略參數，以及**手續費 / 滑價（bps，基點）**。
  4. （選用）展開「參數掃描」：選 1–2 個參數掃出**熱力圖**，看哪一區比較穩（上限 400 組）。
- **你會看到什麼**：
  - **重點數字**：總報酬、年化報酬（CAGR）、夏普值、最大回撤、勝率、交易次數。
  - **權益曲線 + 回撤圖**（$1 成長）。
  - **ATR 交易設定面板**（進場/停損/目標/風險報酬比）。
  - **可下載完整 quantstats 報表（HTML）**。
- **誠實守則（很重要）**：
  - 一律**計入手續費與滑價**；不做「無摩擦」的假回測。
  - **用隔日開盤價成交**，不是用產生訊號當天的收盤價（避免偷看未來）。
  - 所有回測數字請視為**樂觀上限**，實盤通常更差。

### 🧬 多因子選股（Factors，文藝復興視角）

兩個子分頁：

- **Ranking（目前排名）**
  - **用途**：把整個股票池用「價值 / 品質 / 動能 / 成長」四大因子打成 0–100 綜合分數並排名。
  - **怎麼用**：用四條權重滑桿調整各因子比重，表格即時重排。
  - **你會看到什麼**：綜合分數（進度條呈現）＋各分項分數＋本益比、ROE、近半年報酬等。
  - **因子組成**：價值＝本益比/股價營收比（越低越好）＋自由現金流殖利率；品質＝ROE/淨利率/毛利率/低負債；
    動能＝近 3/6/12 月報酬；成長＝營收年增。各因子在「同一天的股票池內」做橫斷面標準化，再轉百分位。
- **Portfolio backtest（因子投資組合回測）**
  - **用途**：定期換股，只持有綜合分數前 N 名，看長期績效，並和「等權重買整池」基準比較。
  - **怎麼用**：設定起始年份、再平衡頻率（月/季）、持股數 N、手續費、因子權重，按下執行。
  - **你會看到什麼**：因子組合的 CAGR/夏普/最大回撤、與基準的差距、**資訊係數 IC**、權益曲線、
    以及各分位的「未來報酬」長條圖（理想是由低到高遞增）。
  - **誠實守則**：
    - 換股當天只用「**當時已公布**」的資料來選股（時點正確，不偷看未來財報）。
    - **存活者偏差警告**：因為股票池是「今天還活著的贏家」，回測會偏樂觀——畫面上有紅字提醒，請當成**上限**。

### ⚖️ 風險（Risk，橋水視角）

- **用途**：體檢單一標的的風險，並和大盤比較。
- **怎麼用**：輸入標的與基準（預設 `SPY.US`），免費資料即可。
- **你會看到什麼**：年化波動率、Beta、與基準的相關性、最大回撤、**VaR 95% / CVaR 95%**（單日尾端風險）、
  夏普值、流動性等級，以及「衰退壓力測試」（= Beta × 大盤 −30% 衝擊的示意估計）。

### 📰 財報（Earnings，摩根大通視角）— 需 `FMP_API_KEY`

- **用途**：財報前布局、看共識預期與歷史驚喜。
- **你會看到什麼**：下次財報日、下季共識 EPS、歷史「優於預期比率」、近期驚喜幅度、近幾季實際 vs 預估。
- **注意**：分析師共識估計屬**付費資料**，需設定 `FMP_API_KEY`；未設定時顯示提示。

### 🌐 總經（Macro，Two Sigma 視角）— 需免費 `FRED_API_KEY`

- **用途**：先看清楚當前的總經環境，再操作個股。
- **你會看到什麼**：關鍵 FRED 指標（CPI、失業率、聯邦基金利率、10Y–2Y 利差、10 年期殖利率、實質 GDP）的
  最新值與年變化、自動產生的訊號（殖利率倒掛、通膨偏高、失業上升、政策偏緊），以及一句「景氣循環階段」研判。

### 🔄 產業輪動（Rotation，城堡視角）

- **用途**：判斷資金該進攻還是防守、哪些產業領先。
- **怎麼用**：按「Run rotation scan」抓取 11 檔 SPDR 行業 ETF（免費資料）。
- **你會看到什麼**：依 1/3/6 月相對強弱排名、進攻 vs 防守傾向（offense/defense）、領先/落後者、分數長條圖。

### 🧺 ETF 投資組合（ETF Portfolio，先鋒視角）

- **用途**：用 ETF 算出一組「效率前緣」配置。
- **怎麼用**：輸入 ETF 清單與方法（**最大夏普** 或 **最小波動**），按「Optimize」（免費資料）。
- **你會看到什麼**：各 ETF 權重（長條圖）、預期報酬、波動、夏普值。
  > 提醒：用歷史估出來的權重很「雜訊」，當成起點而非定論。

> 以上 5 個頁面（以及基本面、技術面）都附**選用 AI 報告**按鈕（見第 4 節）。

## 3. 背後的核心觀念（為什麼可以信任這些數字）

這個工具最在意的是「**不要自己騙自己**」。三個關鍵設計：

1. **時點正確（point-in-time）**：財報一律用「**實際公布日**」當作可得時間，不是用會計期末日。
   例如 6 月底結算、8 月才公布的財報，在 7 月是**不可知的**，工具不會拿來用。
2. **回測誠實**：一定算手續費與滑價、用隔日開盤成交、附最大回撤與報表，不只報好看的年化報酬。
3. **資料分層、來源可換**：所有功能只認一套「**正規化欄位**」與「`代號.市場`」格式；換資料商（例如未來加
   FMP 付費源、或台股 FinMind）只是替換一個 provider，畫面與邏輯都不用改。

## 4. （選用）AI 研究報告

- 基本面、技術面兩個儀表板上各有一個 **🤖 AI report** 摺疊區，可把「**已經算好的數字**」交給 Claude，
  生成一篇研報式的文字（高盛基本面 / 摩根士丹利技術面）。
- **這是選用、可關閉的功能**：核心計算完全不依賴它。需要時才裝 `personas` 套件並在 `.env` 設
  `ANTHROPIC_API_KEY`；沒設定時畫面只會顯示提示，其他功能照常運作。
- **AI 只負責把給定的數字寫成文章，不會自己捏造資料**；缺數字它會明講。

## 6. 誠實的限制（請務必知道）

- **回測是樂觀上限**：即使乾淨的回測也會高估實盤；任何策略上線前請先紙上/前向測試。
- **存活者偏差**：目前用「今天還在的股票」當池子，會高估報酬、低估回撤（多因子頁面有明確警告）。
- **yfinance 非官方**：可能無預警壞掉或限流，不適合當正式基礎。
- **EDGAR 只有美股**：基本面儀表板僅支援有在美國申報的公司（例如 `AAPL.US`、`MSFT.US`、`KO.US`）。
- **多類股權結構**：少數公司（如 META 多類別股本、JNJ/PG 的權益科目）某些欄位會抓不到而顯示 NaN，
  會被相關篩選自動排除——這是已知限制，不是錯誤。
- **AI 報告**：需自備 `ANTHROPIC_API_KEY` 與網路，且會計費。
- **台股**：架構已預留（`代號.TW`／FinMind），但**尚未實作**（Phase 6）。
